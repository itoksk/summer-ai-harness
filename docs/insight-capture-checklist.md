# Insight capture checklist / 気づきを残すチェックリスト

A summary of a conversation is not enough. The harness wants your *thinking*, not just the AI's answer.
Before the day reflection issue is closed, check that at least one of these is written in your own words.

会話の要約だけでは足りません。ハーネスが残したいのは、AIの答えではなく、あなたの **思考** です。
1日の振り返りIssueを閉じる前に、次のどれか1つ以上が **自分の言葉** で書けているか確認しましょう。

- [ ] "At first I thought A, but after checking B, I now think C." / 「最初はAと思ったが、Bを確かめてCと考えるようになった」
- [ ] "In this problem you have to separate A from B." / 「この問題ではAとBを分けて考える必要がある」
- [ ] "Next time I see this kind of problem, I'll check A, then B, then C." / 「次に同じ型の問題を見たら、A→B→Cの順で確認する」
- [ ] "This is still just my guess — I still need to confirm A." / 「これはまだ予想で、Aの確認が必要」

## Good record vs weak record / よい記録・弱い記録

| Weak / 弱い | Good / よい |
| --- | --- |
| "I learned about embeddings." / 「埋め込みを学んだ」 | "I thought similar words would be far apart, but cosine similarity showed they sit close together." / 「似た言葉は遠いと思ったが、コサイン類似度で近いと分かった」 |
| "The AI fixed my bug." / 「AIがバグを直した」 | "The bug was a wrong index. Next time I'll print the variable before assuming." / 「原因はインデックス間違い。次は決めつけず先に変数を表示する」 |

A good record explains not just *what* you found, but *why* the new view makes more sense.
よい記録は「何を見つけたか」だけでなく「なぜ新しい見方の方が筋がよいか」まで書きます。
