# Day map / 日程一覧

The 5 days of the course, each with its title, focus, and the essential questions you'll reflect on.
You write **one reflection issue per day** — and you don't make it by hand: in vibe-local you say
**"start today's reflection"** / **「今日の振り返りを始めたい」** and it creates the issue and
interviews you, using that day's questions below as reference.

コース5日間それぞれのタイトル・核・振り返りで使う「問い」の一覧です。
振り返りは **1日＝1Issue**。手作業では作りません。vibe-local で **「今日の振り返りを始めたい」** と
言えば、下のその日の問いを参考にしながら、Issueを作ってインタビューしてくれます。

> Behind the scenes, vibe-local runs:
> 裏で vibe-local が実行します:
>
> ```bash
> npm run new:day -- --day 2
> ```

---

## Day 1 — AI Playground / 1日目 AIプレイグラウンド

**Focus / 核:** Make your first AI things; notice that the same words give different results. / 初めてAIで作る。同じ言葉でも結果が違うことに気づく。

Essential questions / 問い:

- If everyone types the same words, why does each person get something different? / 全員が同じ言葉を打つのに、なぜ一人ひとり違うものが出るのか?
- If an AI made the picture from your words, whose creation is it really? / AIが君の言葉から作った絵は、本当は誰の作品なのか?
- Can you direct several AIs into one piece that feels truly yours — and put it where the whole world can see it? / 複数のAIを演出して本当に自分のものと言える作品を作り、それを世界に見せられるか?

## Day 2 — Language AI / 2日目 言語AI

**Focus / 核:** See how words become numbers, and read a 200-line GPT — no magic, just code. / 言葉が数字になる仕組みと、200行のGPTを読む。魔法ではなくコード。

Essential questions / 問い:

- How can a computer, which only knows numbers, possibly understand words? / 数字しか分からないコンピュータが、どうやって言葉を理解できるのか?
- Is there any magic inside an AI — or is it just code you could read? / AIの中に魔法はあるのか — それとも、ただの読めるコードなのか?
- Once an AI is made of YOUR words, who controls it — and how safe is it? / AIが「自分のことば」でできたとき、誰がそれを握り、どれだけ安全なのか?

## Day 3 — Data & Machines That See / 3日目 データと「見る」機械

**Focus / 核:** Turn people into data, teach a machine to see, and decide when to trust the math. / 人をデータにし、機械に見せ、数学を信じるか自分で決める。

Essential questions / 問い:

- If a person can be turned into a row of numbers, can math see who you are most like? / 人が数字の並びになるなら、数学は「誰に一番似ているか」を見抜けるのか?
- How does a machine learn to SEE — and what is it really looking at when it gets fooled? / 機械はどうやって「見る」ことを学ぶのか — そして騙されるとき、本当は何を見ているのか?
- When the math gives you an answer, how do you know whether to TRUST it — and when do YOU decide instead? / 数学が答えを出したとき、それを信じてよいかどう見極め、いつ「自分」が決めるのか?

## Day 4 — Vibe Coding / 4日目 バイブコーディング

**Focus / 核:** Build a real app with AI while staying in the driver's seat. / 運転席に座ったまま、AIと本物のアプリを作る。

Essential questions / 問い:

- How do you stay in the driver's seat while an AI writes the code for you? / AIにコードを書かせながら、どうやって自分が運転席に座り続けるのか?
- Now that you can build something real, what does it mean to build it responsibly and finish it? / 本物を作れるようになった今、それを責任をもって作り、やり切るとはどういうことか?

## Day 5 — Build & Demo Day / 5日目 完成とデモデイ

**Focus / 核:** Finish, rehearse, present — and decide who you become after the course. / 仕上げ、リハーサル、発表。そしてコース後の自分を決める。

Essential questions / 問い:

- With time almost gone, do you add more, or make what you already have unforgettable? / 残り時間がわずかな今、足すのか、それとも今あるものを忘れられないものにするのか?
- Now that you can build AI, who do you become when the course ends? / AIを作れるようになった今、コースが終わったとき君は何者になるのか?
